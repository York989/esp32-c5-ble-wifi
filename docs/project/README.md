# 项目文档索引

这个目录用于持续维护当前 BLE 项目的项目文档。它和 `docs/ble-learning/` 不同：这里记录项目本身的设计、配置、运行方式和迭代变化；学习目录记录 BLE 知识路线和学习笔记。

远程仓库：`https://github.com/York989/esp32-c5-ble-wifi.git`

## 建议文档结构

1. `00-项目概览.md`：项目目标、功能范围、当前状态。
2. `01-硬件与环境.md`：开发板型号、ESP-IDF 版本、串口、外设连接。
3. `02-构建烧录运行.md`：构建、烧录、监视日志和常用命令。
4. `03-BLE协议设计.md`：设备名、广播内容、GATT 服务、特征和 UUID。
5. `04-软件架构.md`：任务划分、组件边界、事件流和关键模块。
6. `05-调试记录.md`：问题现象、定位过程、解决方案和复现条件。
7. `06-迭代日志.md`：按时间记录重要功能、配置和行为变化。
8. `07-AI工具与MCP.md`：记录 ESP-IDF MCP、Codex skill 和 AI 辅助开发工具。
9. `08-BLE调试工具.md`：记录 WSL2 内可用的 BLE 调试工具、限制和安装命令。

硬件资料入口：`../data/README.md`，包含开发板用户手册、原理图和整理后的硬件笔记。

## 维护规则

- 项目行为发生变化时，先判断是否需要同步更新这里的文档。
- BLE 服务、Characteristic、UUID、广播字段变化后，必须更新协议设计文档。
- 构建命令、板卡型号、串口或 ESP-IDF 版本变化后，必须更新环境与运行文档。
- 原理图、用户手册、排针定义或硬件注意事项变化后，必须同步更新 `docs/data/`。
- 调试结论要写清楚现象、原因、解决方式，避免只写结果。
