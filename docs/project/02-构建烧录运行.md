# 构建、烧录与调试

## 当前目标

- 芯片目标：`esp32c5`
- 构建目录：`build/`
- 串口：`/dev/ttyACM0`
- ESP-IDF 路径：`/home/u888/esp/.espressif/v6.0.1/esp-idf`

## 常用命令

在 ESP-IDF 环境初始化后运行：

```bash
. /home/u888/esp/.espressif/v6.0.1/esp-idf/export.sh
idf.py build
idf.py -p /dev/ttyACM0 flash monitor
```

## compile_commands.json

主工程的编译数据库位于：

```text
build/compile_commands.json
```

VS Code 中的 `${config:idf.buildPath}/compile_commands.json` 当前解析为：

```text
${workspaceFolder}/build/compile_commands.json
```

`build/bootloader/compile_commands.json` 是 bootloader 的编译数据库，不是应用主工程优先使用的文件。

## OpenOCD

ESP32-C5 当前使用内置 USB-JTAG 配置：

```text
board/esp32c5-builtin.cfg
```

已验证 OpenOCD 可以识别开发板的 USB-JTAG、JTAG tap 和 RISC-V core，并启动 GDB server 端口 `3333`。如果 VS Code 中 OpenOCD 报错，优先检查 `.vscode/settings.json` 中的 `idf.openOcdConfigs`、`idf.adapterTargetName` 和 ESP-IDF 环境是否已正确加载。
