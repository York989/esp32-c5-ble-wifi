# E101-C5WN8 开发板用户手册整理

来源 PDF：`E101-C5WN8系列开发板+datasheet_CN_v1.0 (1).pdf`

## 文档定位

这份整理用于后续 ESP-IDF BLE/Wi-Fi 项目开发，不替代原始用户手册。原始手册共 9 页，主要包含模块介绍、组件说明、管脚定义、供电方式和版本信息。

## 板卡与模块

- 开发板：E101-C5WN8-PS-TB / E101-C5WN8-XS-TB 系列。
- 模组：E101-C5WN8-PS 或 E101-C5WN8-XS。
- 尺寸：约 `52.9 mm * 25.4 mm`，包含 USB 接头，不包含模块天线尺寸。
- 供电接口：USB，也可通过排针供电。
- 通信接口：TTL UART。
- 工作温度：`-40` 到 `+85` 摄氏度。
- 工作湿度：`10%` 到 `90% RH`，无冷凝。
- 储存温度：`-40` 到 `+125` 摄氏度。

E101-C5WN8-PS 支持 2.4 GHz / 5 GHz 双频 Wi-Fi 6、Bluetooth 5 LE、Zigbee 和 Thread。PS 版本使用板载 PCB 天线，XS 版本为 IPEX-1 天线版本。

## 主要组件

| 组件 | 开发意义 |
| --- | --- |
| E101-C5WN8-PS / XS 模组 | 核心无线模组，承载 ESP32-C5 功能。 |
| 两侧 Pin Header | 引出大部分可用 GPIO，flash SPI 总线相关管脚除外。 |
| 5 V 到 3.3 V DC/DC | 为模组和板上 3.3 V 外设供电。 |
| 3.3 V Power On LED | USB 供电后亮起，用于判断 3.3 V 是否存在。 |
| USB-to-UART Bridge | 用于 UART 下载、串口日志和通信，速率最高约 1 Mbps。 |
| ESP32-C5 USB Type-C Port | 原生 USB 口，可供电、烧录、USB 通信和 JTAG 调试；支持 USB 2.0 Full Speed，最高 12 Mbps，不支持 480 Mbps High Speed。 |
| Boot Button | 与 Reset 配合进入固件下载模式。 |
| Reset Button | 复位芯片。 |
| USB Type-C to UART Port | 通过 USB 转 UART 桥接器连接 ESP32-C5，也可供电和烧录。 |
| RGB LED | 可寻址 RGB LED，由 `GPIO27` 驱动。 |
| 电流测量跳帽 | 移除跳帽后可串入电流表测量模组电流。 |

注意：手册组件图中出现过 `ESP32-S3 USB Port` 字样，但正文、原理图和本项目目标均指向 ESP32-C5。这里按 ESP32-C5 处理，并保留该疑点。

## 供电方式

手册列出三类供电方式：

1. 使用 USB Type-C 转 UART 接口或 ESP32-C5 原生 USB Type-C 接口供电。可任选其一，也可同时供电；这是默认推荐方式。
2. 使用 `5V` 与 `GND` 排针供电。
3. 使用 `3V3` 与 `GND` 排针供电。

开发时优先使用 USB 供电。通过 `3V3` 排针外部供电时，需要确认外部电源稳定、纹波和电流能力满足模组需求，避免同时与板载 3.3 V 电源冲突。

## 电流测量

手册说明板上电流测量跳帽可用于测量 E101-C5WN8-PS 模组电流：

- 移除跳帽：开发板外设与模组电源断开，可在跳帽位置串入电流表测量模组电流。
- 安装跳帽：恢复出厂默认连接，开发板正常运行。

原理图中电流跳帽位置与手册丝印命名可能不完全一致，实际测量前应以实物丝印和万用表通断检查为准。

## 开发注意事项

- `GPIO27` 已用于板载 RGB LED，本项目当前 `sdkconfig.defaults.esp32c5` 也配置为 `CONFIG_BLINK_GPIO=27`。
- `GPIO13` / `GPIO14` 是 ESP32-C5 原生 USB D- / D+，如果使用原生 USB、USB-JTAG 或 USB CDC，需要避免复用冲突。
- `GPIO11` / `GPIO12` 对应 UART0 TX/RX，经 USB-to-UART 桥接器使用。
- `MTMS`、`MTDI`、`GPIO7`、`GPIO25`、`GPIO26`、`GPIO27`、`GPIO28` 是 ESP32-C5 strapping 管脚。上电和复位期间外部电平会影响启动配置，外接电路必须谨慎。
- 集成 SPI PSRAM 的模组中，`NC/IO15` 可能被用作 `SPICS1`，外部不可使用；未集成 SPI PSRAM 时可作为 `GPIO15` 使用。

## 对当前项目的直接影响

- BLE 广播和 GATT 功能不依赖外部排针，但 RGB LED 控制会占用 `GPIO27`。
- OpenOCD/USB-JTAG 调试应优先使用 ESP32-C5 原生 USB Type-C 口。
- 串口日志和 UART 烧录可使用 USB Type-C to UART 口。
- 后续加入低功耗测试时，需要利用电流测量跳帽，并记录跳帽状态、供电口和测量仪表连接方式。
